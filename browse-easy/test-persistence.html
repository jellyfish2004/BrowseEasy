BrowseEasy Settings Persistence Test

How to Test Settings Persistence:

  1. Open the BrowseEasy extension panel (click the extension icon)
  2. Go to the Settings tab and enable some accessibility features
  3. Refresh this page - the settings should persist and be automatically applied
  4. Try opening this page in a new tab - settings should also apply there
  5. Use the AI chat to enable features and check if they persist

How to Test Hover Highlighting:

  1. Enable "Highlight on Hover" in the Settings tab or ask the AI: "highlight elements when I hover"
  2. Move your cursor over the interactive elements below (buttons, links, inputs)
  3. You should see a blue background with dashed blue outline appear when hovering
  4. Non-interactive elements (spans, divs, disabled elements) should NOT highlight
  5. The highlighting should disappear when you move the cursor away
  6. Note: This is different from "Highlight Links" which shows permanent yellow highlighting

Test Instructions

Expected Behavior:
  • Settings should persist when you refresh the page
  • Settings should apply automatically when opening new tabs
  • Changes made in the Settings tab should be saved
  • Changes made via AI chat should be saved
  • Settings should sync across all tabs
Hover Highlighting Expected Behavior:
  • Only interactive elements should highlight on hover (buttons, links, inputs, etc.)
  • Disabled elements should NOT highlight
  • Empty links (href="" or href="#") should NOT highlight
  • Regular text, spans, and divs should NOT highlight
  • Highlighting should appear immediately when cursor enters element
  • Highlighting should disappear when cursor leaves element
  • The highlight should be a blue background with dashed blue outline
  • Should NOT show permanent highlighting - only on actual hover
  • Different from "Highlight Links" feature (which is yellow/permanent)
